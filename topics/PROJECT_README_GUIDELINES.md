# Project README Guidelines (From GitHub Template)

このドキュメントは、**GitHub テンプレートリポジトリから新規プロジェクトを作成した直後**に、README をどう整備するかを定める。

テンプレートリポジトリ自体の README 設計については [TEMPLATE_README_GUIDELINES.md](TEMPLATE_README_GUIDELINES.md) を参照すること。

---

## 1. Initial Customization After Creating from Template

テンプレートから作成した直後に以下を実施する。

### 1.1 Placeholder Replacement

テンプレートに含まれるプレースホルダをすべて実際の値に置き換える。置換漏れがないかチェックリスト（§4）で確認する。

| プレースホルダ | 置き換える内容 |
|---|---|
| `[PROJECT_NAME]` | 実際のプロジェクト名 |
| `[REPO_NAME]` | GitHub リポジトリ名 |
| `[YEAR]` | 著作権表記の年（例: 2025） |
| `[AUTHOR]` | 著者名または組織名 |
| `[user]` / `[org]` | GitHub ユーザー名 / 組織名 |

### 1.2 Meta Information Table

`README-jp.md` のメタ情報テーブルをプロジェクト固有の内容に更新する。

| 項目 | 更新内容 |
|---|---|
| 開発対象 | このプロジェクトで作るもの（テンプレートの説明ではない） |
| 開発環境 | 実際のチーム規模・体制 |
| 主言語 | 実際に使用する言語 |
| AI ツール（任意） | 使用するツールのみ記載、未使用は削除 |

### 1.3 One-Line Summary

「このテンプレートを使った〜」という説明をプロジェクト固有の説明に書き換える。

### 1.4 Feature List

テンプレートが列挙していた「テンプレートとしての特徴」をプロジェクトの実際の特徴に置き換える。

---

## 2. Sections to Remove

テンプレートリポジトリ固有のセクションはプロジェクト側 README には不要なため削除する。

| 削除対象セクション | 理由 |
|---|---|
| テンプレートの使い方（クローン手順） | プロジェクト固有のクイックスタートに置き換える |
| テンプレートのカスタマイズ手順 | 初回セットアップ完了後は不要 |
| 「このテンプレートを使う」ボタンへの言及 | プロジェクトリポジトリには不要 |

---

## 3. Ongoing Maintenance

### 3.1 When to Update

以下のタイミングで README（日本語版・英語版）を同一コミットで更新する。

- 主要な機能追加・変更
- 動作要件・依存関係の変更
- プロジェクト構造の大幅な変更

### 3.2 Bilingual Consistency

言語ポリシーは [LANGUAGE_POLICY.md](../LANGUAGE_POLICY.md) に従う。

- 日本語版（`README-jp.md`）を正本として編集し、英語版（`README.md`）をそれに合わせて更新する
- 両ファイルは同一コミットで更新する

---

## 4. Validation Checklist

プロジェクト開始時（テンプレートからの作成後）に確認する。

```
[ ] プレースホルダ（[PROJECT_NAME]、[YEAR]、[AUTHOR] 等）の置換漏れがないか
[ ] メタ情報テーブルがプロジェクト固有の内容になっているか
[ ] 一行概要がテンプレート説明ではなくプロジェクト固有の説明になっているか
[ ] 特徴セクションがテンプレートの特徴ではなくプロジェクトの特徴になっているか
[ ] テンプレート固有セクション（§2）を削除したか
[ ] LICENSE ファイルの [YEAR] と [AUTHOR] を更新したか
[ ] CI バッジの URL がこのリポジトリを指しているか
[ ] ドキュメント索引のリンクが実在するファイルを指しているか
[ ] 日本語版・英語版が同期しているか
```
